OCR Automation

Intelligent Document & Invoice Extraction Pipeline

An end-to-end document automation system that extracts structured data from scanned invoices and PDFs — vendor names, totals, line items — and writes them directly to Google Sheets, processing 500+ documents daily with 97% field accuracy.

Role

AI/Automation Engineer

Timeline

2 months

Stack

Python, PaddleOCR, LayoutLMv3, Google Sheets API, Docker

Problem

The client's accounts payable team manually keyed data from 500+ invoices per day into spreadsheets. Invoices arrived in varying formats — scanned PDFs, photos, and digital PDFs — from over 200 vendors. Manual entry caused a 15% error rate and consumed 3 full-time employees' time.

Approach

I built a four-stage pipeline optimized for diverse invoice layouts:

  • Preprocessing: Scanned images are deskewed, binarized, and enhanced for OCR quality. PDFs are rendered to high-DPI images.
  • OCR: PaddleOCR extracts raw text with bounding box coordinates, handling tables, multi-column layouts, and low-quality scans.
  • Layout Understanding: LayoutLMv3 classifies each text region into semantic fields — vendor name, invoice number, date, line items, subtotals, tax, and total — using the spatial relationship between text blocks.
  • Export: Extracted fields are validated, normalized, and pushed to Google Sheets via the API in real time, with a confidence-flagging system for human review of low-confidence extractions.

Outcome

The system achieved 97% field-level accuracy across all vendor formats after a 2-week fine-tuning period. It eliminated 2.5 FTEs of manual data entry work and reduced invoice processing turnaround from 48 hours to under 2 hours.

Key Technologies

  • PaddleOCR — High-accuracy multilingual OCR engine
  • LayoutLMv3 — Document layout understanding and field classification
  • Google Sheets API — Zero-touch data export
  • Docker — Containerized deployment
  • Python — Pipeline orchestration and data validation
