Real-Time Object Counting & Tracking Dashboard

A live monitoring system built for a retail client that tracks and counts people across multiple camera zones, delivering real-time heatmaps, dwell-time analytics, and crowd density alerts — reducing manual monitoring overhead by 70%.

Role

Backend & CV Engineer

Timeline

2.5 months

Stack

Python, YOLOv8, DeepSORT, FastAPI, WebSocket, Redis

Problem

The client operated 12 retail locations with over 40 cameras. Security staff manually monitored feeds to estimate foot traffic, identify crowded zones, and flag capacity issues — a process that was labor-intensive, error-prone, and provided no historical analytics.

Approach

I built an end-to-end pipeline from camera ingestion to dashboard visualization:

  • Detection: YOLOv8 processes each camera feed at 30 FPS, detecting and classifying people with bounding boxes.
  • Tracking: DeepSORT assigns persistent IDs to individuals, enabling accurate counting even with occlusion, re-entry, and cross-camera handoff.
  • Analytics: A real-time engine computes zone-level counts, dwell-time distributions, and density heatmaps, pushing updates via WebSocket every 500ms.
  • Alerting: Configurable thresholds trigger crowd density alerts to store managers via push notifications and dashboard banners.

Outcome

The dashboard replaced 6 full-time monitoring staff across all locations, reducing overhead by 70%. Zone dwell-time analytics enabled the client to optimize store layout, resulting in a 12% increase in average transaction value within the first quarter.

Key Technologies

  • YOLOv8 — Real-time person detection
  • DeepSORT — Multi-object tracking with re-identification
  • FastAPI + WebSocket — Low-latency backend and live data streaming
  • Redis — In-memory state management for zone counters
  • HTML Canvas — Real-time heatmap visualization
